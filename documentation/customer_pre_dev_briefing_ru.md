## 1. Юридический блокер: лицензия Element X

### Суть проблемы

Element X iOS распространяется под лицензией **AGPL v3** — это лицензия с открытым исходным кодом, которая требует, чтобы пользователи могли свободно копировать и распространять приложение. Однако правила Apple App Store это запрещают: Apple контролирует установку и распространение, накладывает защиту от копирования (DRM).

**Это означает:** простая публикация нашего форка Element X в App Store создаёт **юридическое противоречие** между лицензией и правилами магазина.

### Чем это грозит

- Компания Element (правообладатель) может предъявить юридические претензии
- Apple может снять приложение с публикации
- Заказчик и разработчик несут репутационные и юридические риски

### Что нужно сделать

Для легальной публикации необходимо получить **коммерческую лицензию** от компании Element (New Vector Ltd). Это отдельное соглашение, которое разрешает распространение через App Store без ограничений AGPL.

### Действия с вашей стороны

- **Решить**, кто инициирует контакт с Element — вы или я (готов сделать это от вашего имени или совместно)
- **Понимать**, что стоимость лицензии — отдельная статья расходов, не входящая в стоимость разработки
- **Учитывать**, что ответ от Element может занять **1–3 недели**
- **Определить запасной вариант**, если Element откажет или запросит неприемлемую сумму

> **Без решения этого вопроса публикация в App Store невозможна.** Разработка может идти параллельно, но выпуск приложения будет заблокирован до получения лицензии.

## 2. Расхождения между ТЗ и реальным проектом

При анализе кодовой базы Element X iOS выявлены существенные отличия от того, что описано в техническом задании. Эти расхождения требуют ваших решений.

### 2.1. Push-уведомления: Firebase отсутствует

- **В ТЗ:** push-уведомления через Firebase Cloud Messaging (FCM)
- **В реальности:** Element X iOS использует **APNs** (стандартная система Apple) напрямую. Firebase SDK в проекте нет вообще.

**Что это значит для вас:**
- APNs — это стандартный, нативный и рекомендуемый способ отправки push на iOS
- Firebase на iOS всё равно передаёт уведомления через APNs — это просто дополнительная прослойка
- Добавление Firebase потребует **+10–16 часов** дополнительной работы и **+$170–$270** к бюджету
- Ваш Sygnal push-gateway должен быть настроен для работы с APNs (это штатная возможность Sygnal)

**Рекомендация:** использовать APNs. Быстрее, проще, надёжнее, дешевле.

**Нужно ваше решение:** согласны ли вы на APNs вместо FCM?

### 2.2. Звонки: Jitsi не поддерживается

- **В ТЗ:** групповые звонки через Jitsi
- **В реальности:** Element X iOS использует **Element Call** — другую систему, основанную на LiveKit SFU

**Что это значит для вас:**
- Jitsi и Element Call — это разные, несовместимые технологии
- В коде Element X iOS нет поддержки Jitsi, и добавление её — это полноценная разработка новой функции, выходящая за рамки форка
- Для работы звонков на вашей стороне необходим **сервер LiveKit SFU** и **развёртывание Element Call**

**Нужно ваше решение:**
- Если у вас есть возможность развернуть LiveKit — звонки будут работать без изменений в коде
- Если у вас только Jitsi — звонки в первой версии работать не будут. Можно отложить их на v2.

### 2.3. Минимальная версия iOS: 16 невозможна

- **В ТЗ:** поддержка iOS 16+
- **В реальности:** Element X iOS требует минимум **iOS 17.0**

**Что это значит для вас:**
- Это ограничение самого проекта Element X, его нельзя обойти без полной переработки кода
- iOS 17+ поддерживается на iPhone 8 и новее — это более 90% активных устройств
- Практическая потеря аудитории минимальна

**Нужно ваше решение:** подтвердите, что iOS 17+ приемлемо.

## 4. Реалистичность сроков: 4–6 недель

### Честная оценка

| Сценарий | Срок | Условия |
|----------|------|---------|
| **4 недели** | Возможен, но маловероятен | ВСЕ блокеры решены заранее. Бэкенд полностью готов с Дня 1. Дизайн-материалы готовы. App Store одобряет с первой попытки. Никаких технических проблем. |
| **6 недель** | Реалистичный сценарий | Большинство вопросов решены к началу работ. Бэкенд готов к Неделе 2. 1 цикл отклонения App Store. Нормальные технические трудности. |
| **8 недель** | С запасом | Задержки с лицензированием или инфраструктурой. 2–3 цикла отклонения App Store. |

### Почему App Store — непредсказуемый фактор

Apple проверяет каждое приложение перед публикацией. Для мессенджеров проверка **особенно строгая**:

- **Guideline 4.3 (Design Spam)** — Apple может отклонить приложение как «копию существующего» (Element X уже в App Store). Мы должны убедительно показать, что наше приложение — не спам, а отдельный продукт для отдельной аудитории.
- **Модерация контента** — Apple требует наличие функций «Пожаловаться» и «Заблокировать пользователя». Они есть в Element X, но будем проверять работоспособность.
- **Шифрование** — приложение использует E2EE, что требует декларации экспортного соответствия.

Каждый цикл «отклонение → исправление → повторная подача → проверка» занимает **3–7 рабочих дней**.

### Что влияет на сроки больше всего

| Фактор | Влияние | Контролирует |
|--------|---------|-------------|
| Скорость ответа от Element (лицензия) | До +3 недель | Element |
| Готовность вашей серверной инфраструктуры | До +2 недель | Вы |
| Скорость предоставления дизайн-материалов | До +1 недели | Вы |
| Количество циклов ревью App Store | До +3 недель | Apple |
| Скорость ответов на мои вопросы | До +1 недели | Вы |

> **Вывод:** из 4–6 недель примерно **половина зависит от вас и внешних факторов**, а не от скорости разработки. Чем быстрее вы предоставите всё необходимое, тем быстрее мы закончим.

## 6. Что НЕ входит в проект

Во избежание недоразумений фиксирую, что в стоимость и объём работ **не входит:**

- Настройка или обслуживание серверной инфраструктуры (Matrix, Sygnal, LiveKit, TURN/STUN)
- Разработка нового функционала, которого нет в Element X
- Дизайн-работы (вы предоставляете готовые материалы)
- Разработка Android-версии
- Стоимость коммерческой лицензии Element
- Подписка Apple Developer Program ($99/год)
- Поддержка и обновления после публикации (по отдельному соглашению)

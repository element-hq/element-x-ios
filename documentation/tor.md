# Техническое задание на разработку iOS-клиента мессенджера на базе Element X

> **Источник:** `TOR.docx` (конвертировано в Markdown для удобства работы)

---

## 1. Общая часть

Данный документ описывает требования и условия разработки iOS-клиента мессенджера, основанного на открытом исходном коде Element X (официальный open-source Matrix-клиент, реализованный на SwiftUI). Проект предполагает создание мобильного приложения для обмена зашифрованными сообщениями и аудио/видео звонками, интегрированного с существующей инфраструктурой Matrix, принадлежащей заказчику.

В рамках первой версии планируется использовать стандартный функционал и интерфейс Element X без разработки нового backend или существенных изменений UI. Приложение будет подвергнуто лишь базовому брендированию под заказчика — изменены иконка и название приложения, уникальный Bundle ID, а при необходимости незначительно скорректирована цветовая схема в рамках допустимого. Push-уведомления от сервера Matrix должны доставляться через сервис Firebase Cloud Messaging (FCM), требующий интеграции соответствующего SDK в клиент. Готовая сборка приложения по завершении работ должна быть опубликована в App Store от имени аккаунта разработчика-исполнителя, обеспечив доступность приложения конечным пользователям.

---

## 2. Цели проекта

- **Запуск брендированного мессенджера на iOS:** предоставить заказчику готовое к использованию iOS-приложение мессенджера под его брендом, основанное на современном клиенте Element X и полностью совместимое с протоколом Matrix.
- **Интеграция с инфраструктурой заказчика:** обеспечить бесшовную работу приложения с имеющимся у заказчика сервером Matrix (homeserver) и сервисами (включая .well-known конфигурацию, интеграционный сервис Scalar, push-шлюз и сервер видеозвонков Jitsi), без необходимости доработки серверной части.
- **Реализация ключевого функционала:** гарантировать поддержку всех основных функций обмена сообщениями и вызовов — регистрация, авторизация, индивидуальные и групповые чаты, push-уведомления, аудио/видео звонки, управление профилем — на уровне, не уступающем стандартному приложению Element X.
- **Соблюдение требований App Store:** выпустить приложение в соответствии с правилами Apple (технические требования, политика конфиденциальности, использование актуальных SDK), успешно пройти модерацию и опубликовать приложение в магазине для широкой аудитории.
- **Качество и безопасность:** обеспечить высокую стабильность работы клиента и безопасность пользовательских данных (включая сквозное шифрование сообщений), используя проверенные решения из экосистемы Matrix и соблюдая лучшие практики разработки мобильных приложений.

## 3. Область работ

В рамках проекта исполнитель выполняет следующие основные работы:

### 3.1. Адаптация исходного кода Element X

Настройка репозитория и проекта под нового клиента (установка уникального Bundle ID, обновление идентификаторов приложения, настройка схемы сборки). Включает подключение необходимых библиотек (например, Matrix Rust SDK, Firebase SDK для FCM) и конфигураций для целевой платформы iOS.

### 3.2. Брендирование приложения

Замена стандартных ресурсов Element X на бренд заказчика:
- Разработать и интегрировать новую иконку приложения
- Установить требуемое название и описание
- Обновить цветовые акценты UI (если согласовано) без нарушения целостности дизайна

Все изменения брендинга должны соответствовать гайдлайнам Apple (иконка заданных размеров и форматов, уникальное название, не противоречащее правилам App Store).

### 3.3. Интеграция push-уведомлений через FCM

Подключение Firebase Cloud Messaging для iOS:
- Регистрация приложения в Firebase
- Получение Google-Services конфигурации
- Добавление Push Notifications и Background Modes в Capabilities проекта Xcode
- Настройка взаимодействия с push-шлюзом Matrix (Sygnal) таким образом, чтобы уведомления о новых сообщениях и вызовах доставлялись через FCM на устройства пользователя
- Настройка APNs-ключа или сертификата в Firebase для отправки push на iOS
- Проверка получения push-уведомлений при разных состояниях приложения (активно, фоновый режим, выгружено)

### 3.4. Конфигурация серверного окружения

Настройка клиента для работы с серверной инфраструктурой заказчика:
- Указать в настройках приложения URL домашнего сервера (homeserver) заказчика
- При необходимости — URL сервера идентификации (identity server) и других служб
- Убедиться в корректной обработке файла конфигурации `.well-known/matrix/client`
- Убедиться в интеграции с Scalar (если необходимы виджеты/боты)
- Обеспечить корректную работу со шлюзом вызовов Jitsi для групповых звонков
- Все параметры (адреса серверов) либо задаются в настройках приложения, либо вшиваются в сборку согласно инфраструктуре заказчика

### 3.5. Реализация поддержки звонков

Обеспечение работы функции аудио- и видеовызовов:
- В Element X реализована поддержка 1:1 звонков через Matrix RTC; необходимо протестировать на инфраструктуре заказчика (включая p2p-соединение, обмен сигналами через сервер)
- Для групповых звонков/конференций, при наличии у заказчика Jitsi-сервера, убедиться, что приложение может открывать соответствующий интерфейс (встроенный веб-вью или внешнее приложение Jitsi Meet)
- Возможно, понадобится указать URL сервера Jitsi или убедиться, что он доступен через интеграцию Scalar
- Настройка и тестирование звонков между аккаунтами, устранение проблем (разрешения камеры/микрофона, сеть и т.д.)

### 3.6. Тестирование функциональности

Проведение полного цикла тестирования на разных устройствах и версиях iOS:
- Регистрация нового пользователя
- Вход под существующим аккаунтом
- Обмен сообщениями в личном чате и в групповой комнате
- Получение и отправка мультимедиа (изображения, файлы)
- Получение push-уведомлений
- Входящий и исходящий звонок
- Обновление профиля
- Выход из аккаунта
- Сценарии офлайн/онлайн (корректная синхронизация при возобновлении сети)
- Работа приложения при фоновых запусках (push → нажатие → приложение открывается на нужный чат)
- Тестирование на минимально поддерживаемой и последней актуальной версии iOS
- Тестирование на реальных устройствах (iPhone предыдущих поколений + новейшие модели)

### 3.7. Подготовка сборки (Release Build)

- Установка номера версии и номера сборки (1.0.0 или по требованиям заказчика)
- Выключение тестовых режимов/логов
- Проверка наличия всех ресурсов (иконки всех размеров, Launch Screen, локализации)
- Сборка с профилем App Store Distribution и подписание сертификатом Apple Developer Program
- Bitcode отключен (Apple отключил поддержку bitcode)
- Валидация через Xcode Organizer или Transporter без ошибок

### 3.8. Публикация в App Store

Публикация через аккаунт Apple Developer исполнителя:
- Создание записи приложения в App Store Connect
- Заполнение метаданных: название, подзаголовок, описание, ключевые слова, Privacy
- Загрузка IPA в TestFlight для предварительного тестирования
- Предоставление ссылки заказчику для ознакомления
- Отправка на App Review (модерацию Apple)
- В случае отклонения — анализ причин, оперативное исправление, повторная подача
- После одобрения — выпуск в статус Ready for Sale

## 4. Требования к функциональности

Все перечисленные возможности уже присутствуют в Element X и должны быть сохранены/активированы в кастомной версии.

### 4.1. Регистрация пользователей

- Создание нового аккаунта Matrix на сервере заказчика прямо из приложения
- Ввод логина, пароля, выбор сервера (homeserver заказчика по умолчанию)
- Корректная обработка дополнительного подтверждения: капча, email (при условии поддержки Element X)

### 4.2. Авторизация и вход

- Экран логина с вводом Matrix ID или выбором сервера, логин/пароль
- Поддержка OAuth/OIDC, если на стороне сервера настроено SSO
- Аутентификация на homeserver через Matrix API, сохранение токена сессии
- Возможность разлогинивания (Logout), очищение локальных данных, вход под другим аккаунтом

### 4.3. Синхронизация и загрузка данных

- Загрузка списка чатов (комнат), контактов, истории сообщений, ключей шифрования
- Использование механизма **Sliding Sync** для быстрой и эффективной синхронизации
- Мгновенная загрузка последних сообщений и постепенная подгрузка истории по мере прокрутки

### 4.4. Обмен сообщениями

- **Отправка и получение** в 1:1 и групповых чатах в реальном времени
- **История переписки** с временными метками, именами отправителей, аватарами
- **Квитанции о прочтении** (read receipts) — показ, какие сообщения отмечены как прочитанные
- **Индикация набора текста** — индикатор «печатает...» в реальном времени
- **Вложения** — отправка изображений (из галереи/камеры), видео, документов, файлов; просмотр/скачивание вложений. Запрос необходимых разрешений (фото, камера, файлы)
- **Реакции и эмодзи** — emoji-реакции на сообщения (если поддерживается Element X)
- **Управление сообщениями** — удаление собственных сообщений (с опцией «удалить для всех»), отображение отредактированных сообщений, пересылка, цитирование

### 4.5. Управление чатами (комнатами)

- **Список чатов** с отображением непрочитанных сообщений и последних сообщений
- **Создание нового чата** — личный диалог по Matrix ID или из контактов; создание групповой комнаты (название, приглашение участников)
- **Поиск и подключение к комнатам** — через серверный каталог или .well-known; присоединение к открытым комнатам, запрос на присоединение
- **Управление участниками** — просмотр списка, добавление участников, возможность покинуть/удалить/скрыть беседу
- **Информация о комнате** — название, аватар, описание/тема, настройки уведомлений, права модерации

### 4.6. Профиль пользователя

- **Просмотр профиля:** имя пользователя (display name), аватар, статус
- **Редактирование:** изменение отображаемого имени, загрузка нового фото аватара
- **Просмотр Matrix ID** и связанных данных (привязанные email/телефоны)
- **Безопасность:** просмотр устройств (сессий), подписанных ключей, удаление лишних сессий

### 4.7. Аудио- и видеовызовы

#### 1:1 звонки
- Звонок из интерфейса диалога (значок звонка/видео)
- Соединение через Matrix (WebRTC)
- Экран разговора: участники, длительность, кнопки мут/завершение
- Входящий звонок: уведомление/экран вызова, принять/отклонить
- Корректная работа микрофона, динамика, камеры, ориентации экрана, фонового режима

#### Групповые звонки/конференции
- Поддержка через виджет Jitsi — «начать конференцию» в комнате
- Встроенный WebView с Jitsi (настроенным на сервер заказчика) или переадресация на приложение Jitsi Meet
- Участие в групповых звонках без дополнительных сложных настроек

### 4.8. Push-уведомления

- Входящее сообщение (личный/групповой чат) — push с указанием отправителя и фрагмента текста
- Входящий звонок — уведомление/системный экран звонка (CallKit, если поддерживается)
- Нажатие на уведомление открывает приложение на соответствующем чате/экране звонка
- Реализация через FCM: регистрация FCM-токена, передача на сервер Matrix через API pushers
- Бейдж на иконке приложения с количеством непрочитанных сообщений

### 4.9. Внутренние уведомления и фоновые обновления

- Обновление списка чатов и сообщений в реальном времени (Matrix sync / sliding sync)
- Индикаторы нового сообщения в UI (точка у непрочитанных чатов, звук/вибрация)
- Фоновое получение сообщений (периодическая фоновая синхронизация)

### 4.10. Настройки приложения

- **Уведомления:** включение/отключение звука, вибрации, превью сообщений
- **Отображение:** тема (светлая/тёмная), размер шрифта (системные возможности)
- **Сессии и ключи:** просмотр устройств, verification (взаимная проверка), экспорт/восстановление ключей шифрования (Secrets Recovery Key)
- **Прочее:** отправка отчёта об ошибке (Rageshake) и т.д. — могут присутствовать и не требуют отключения

### 4.11. Безопасность и шифрование

- **E2EE по умолчанию** — все сообщения и звонки сквозно зашифрованы (Olm/Megolm)
- Корректный обмен ключами шифрования, безопасное хранение в Keychain
- Индикаторы шифрования (значки замка)
- Защищённое хранение данных на устройстве (кэш, файлы, ключи)
- Локальная блокировка PIN-кодом или биометрией (если есть в Element X)
- Минимальные необходимые разрешения с обоснованием в Info.plist (`NSCameraUsageDescription`, `NSMicrophoneUsageDescription` и др.)

> **Примечание:** Все функциональные требования реализуются за счёт возможностей базового клиента Element X. Новые сценарии, не поддерживаемые Element X из коробки, в v1 не закладываются. Необходимо убедиться, что никаких регрессий или отключённых функций в кастомной сборке не возникло.

## 5. Ограничения

### 5.1. Отсутствие разработки нового функционала в v1

Первая версия — переработка (fork) Element X без создания уникальных функций или экранов. Все требования удовлетворяются существующим функционалом. Дополнительные возможности выносятся за рамки текущего ТЗ.

### 5.2. Стандартный UI/UX

Интерфейс остаётся неизменным относительно стандартного Element X. Брендирование ограничивается косметическими изменениями (цветовая тема, логотип, название). Структура экранов, навигация и логика взаимодействия не меняются.

### 5.3. Backend и серверная часть

Разработка серверной логики не требуется. Заказчик предоставляет настроенную инфраструктуру:

| Компонент | Назначение |
|-----------|-----------|
| **Homeserver** | Основной сервер Matrix (аккаунты, переписка) |
| **`.well-known`** | Автоматическое определение адресов серверов |
| **Identity Server** | Привязка email/телефонов (MatrixIS или аналог) |
| **Push-gateway (Sygnal)** | Пересылка push-уведомлений на FCM/APNs |
| **Integration manager (Scalar)** | Виджеты, боты внутри приложения |
| **Jitsi-сервер** | Видеоконференции |

Если какие-то сервисы недоступны или отключены, соответствующая функциональность на клиенте может быть ограничена (допущение).

### 5.4. Интеграции с внешними системами

Не входят в объём: CRM-системы, внешние БД, собственные push-сервисы, административные панели, веб-интерфейсы управления. Приложение взаимодействует только с Matrix-серверами и службами через стандартные протоколы.

### 5.5. Минимальная версия iOS

**iOS 16.0 и выше** (iPhone и iPod Touch, совместимые с iOS 16+). Обоснование: современные возможности SwiftUI, соответствие требованиям Apple к новым приложениям. Поддержка iOS 15 и старше не планируется.

### 5.6. Совместимость устройств

Основной акцент — iPhone. Приложение универсальное (iOS/iPadOS), можно установить на iPad, но специальная адаптация под iPad не требуется. Приложение должно корректно масштабироваться без критических ошибок.

### 5.7. Лицензирование исходного кода

Element X распространяется под **AGPL v3.0**. При модификации и распространении необходимо соблюдать условия лицензии:
- Исходный код модификаций должен быть доступен пользователям при распространении
- Либо открыть репозиторий, либо рассмотреть приобретение коммерческой лицензии от Element
- Исполнитель консультирует заказчика по вопросам лицензии и обеспечивает, чтобы публикация не нарушала соглашений

### 5.8. Конфиденциальность и политика безопасности

- Все собираемые данные перечислены в декларации App Privacy (App Store Connect)
- Обработка в соответствии с политикой конфиденциальности заказчика
- Никаких скрытых трекеров сверх необходимых для Matrix-клиента
- Firebase может собирать базовую аналитику — отразить в Privacy
- Не использовать недокументированные API, не запрашивать избыточные разрешения

## 6. Сборка и публикация

### 6.1. Средства разработки

Xcode актуальной версии с поддержкой iOS 17 SDK или новее. Оптимизация под современные устройства.

### 6.2. Конфигурация сборки

- Тип: **Release** для App Store
- Bundle ID: уникальный, привязанный к аккаунту исполнителя (отличный от `com.element…`)
- Версионность: `1.0.0 (Build 1)` или по пожеланиям заказчика
- Capabilities: Push Notifications, Background Modes (Remote notifications), Keychain Sharing (если для E2EE), App Groups (при необходимости)
- Firebase настроен для Release-окружения
- Тестовые/дебажные настройки не попадают в сборку

### 6.3. Профилирование и оптимизация

- Профилирование: память, утечки, время запуска
- Отключение debug-логирования
- Плавная работа на минимально поддерживаемом устройстве (iPhone 8 / iOS 16) и актуальных (iPhone 14/15 / iOS 17/18)
- Оптимизация размера: удаление неиспользуемых ресурсов, Bitcode отключен, отладочные символы не включены

### 6.4. Метаданные для App Store

- **Описание** — на русском и английском (или только на требуемых рынках)
- **Название** (Display Name) для App Store и Springboard
- **Скриншоты** — на разных устройствах (5.5" iPhone 8, 6.5" iPhone 11 Pro Max или новее, iPad). Ключевые экраны: список чатов, переписка, звонок, профиль. Без тестовых данных, нарушающих конфиденциальность
- **Иконки** — App Icon всех размеров, включая 1024x1024 PNG для App Store
- **Категория** — «Социальные сети» или «Productivity» (по согласованию)

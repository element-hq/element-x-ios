## 8. Юридические вопросы

### 8.1. Лицензия AGPL v3

Приложение основано на открытом коде Element X iOS (лицензия AGPL v3). Для публикации в App Store необходимо **письменное подтверждение**, что лицензия AGPL v3 покрывает этот форк.

> **Нужно ваше действие:** Предоставьте письменное подтверждение (email или документ) о том, что публикация UCMeet.Chat в App Store допускается в рамках AGPL v3.

**Это блокирующий вопрос** — без этого подтверждения мы не можем отправить приложение на ревью.

### 8.2. Декларация шифрования (Encryption Export Compliance)

Приложение использует сквозное шифрование (E2EE) через Matrix SDK. По правилам BIS (Bureau of Industry and Security, США) требуется ежегодная самоклассификация.

| Параметр | Значение |
|----------|----------|
| Использует шифрование | Да |
| Тип | Сквозное (Olm/Megolm через Matrix Rust SDK) |
| Классификация ECCN | 5D002 |
| Исключение | Массовое распространение (EAR §740.17(b)(1)) |
| `ITSAppUsesNonExemptEncryption` | `true` (уже настроено) |

Для подачи самоклассификации потребуются данные вашего юридического лица:

```
Название продукта: UCMeet.Chat
Версия: 1.0.0
ECCN: 5D002
Тип: Программное обеспечение
Подавшая сторона: [ЗАПОЛНИТЬ — название юридического лица]
Производитель: [ЗАПОЛНИТЬ — название юридического лица]
Описание: Мобильный мессенджер на протоколе Matrix со сквозным
  шифрованием (Olm/Megolm) через Matrix Rust SDK.
Тип авторизации: Mass Market (EAR §740.17(b)(1))
```

> **Нужно ваше действие:** Укажите официальное название юридического лица для подачи самоклассификации.

### 8.3. Конфиденциальность (Privacy Manifest)

Privacy Manifest уже настроен в приложении. В приложении **нет рекламных трекеров**. Аналитика (PostHog, Sentry) — только по согласию пользователя.

## Как отправить ответы

Удобнее всего — скопировать чек-лист выше и заполнить статус каждого пункта. Если нужно внести изменения в тексты — отправьте исправленную версию.

Формат ответа не важен — подойдёт email, сообщение в чате или документ.

**После получения всех подтверждений мы:**
1. Сделаем скриншоты и отправим на одобрение
2. Заполним все поля в App Store Connect
3. Загрузим сборку через Xcode
4. Отправим на ревью Apple

Ожидаемый срок проверки Apple: 1–3 рабочих дня.

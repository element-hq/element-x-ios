## Неделя 1 — Настройка проекта и начало лицензирования (Дни 1–5)

### День 1 — 4 ч | Форк и настройка окружения
**Фаза:** 1 (Настройка проекта)

| № | Задача | Описание |
|---|--------|----------|
| 1 | Сделать форк репозитория Element X iOS | Создать приватный форк от выбранного релизного тега (рекомендуется iOS 17+). Настроить локальный Git-репозиторий со стратегией веток (main + develop). |
| 2 | Отправить запрос о лицензировании в Element | Составить и отправить письмо в Element (New Vector Ltd) о коммерческой лицензии для публикации в App Store. Включить описание масштаба проекта. |
| 3 | Установить XcodeGen | Установить через Homebrew (`brew install xcodegen`), проверить совместимость версии с `project.yml` проекта. |
| 4 | Начать разрешение зависимостей SPM | Запустить `xcodegen generate`, открыть сгенерированный `.xcodeproj`, позволить Xcode разрешить все зависимости Swift Package Manager. Это может занять 30–60 минут. |

**Ожидаемый результат:** Репозиторий форкнут, запрос о лицензировании отправлен, XcodeGen установлен, разрешение SPM начато.

**Ожидаемый результат:** Немодифицированный Element X собирается и запускается на симуляторе. Окружение сборки задокументировано.

## Недели 7–8 — Буфер: циклы ревью App Store (Дни 31–40)
**Фаза:** 9 (Циклы ревью — продолжение)
**Часы:** До 40 ч (используются только при необходимости)

### Назначение

Процесс ревью App Store непредсказуем, особенно для:
- **Мессенджеров, впервые публикуемых** (пристальное внимание к модерации контента)
- **White-label / форк-приложений** (Guideline 4.3 — Design Spam)
- **Приложений с шифрованием** (экспортный контроль)

Ожидайте **1–3 цикла отклонения/повторной отправки**, каждый занимает 2–5 рабочих дней на ревью Apple.

### Типичные причины отклонения и план реагирования

| Причина отклонения | Вероятность | Время реагирования | План действий |
|-------------------|------------|-------------------|---------------|
| **Guideline 4.3 — Design Spam** | Высокая | 4–8 ч | Усилить отличия: уникальное описание, выраженная визуальная идентичность, чёткое бизнес-обоснование в заметках для ревью. Апелляция при необоснованности. |
| **Недостаточная модерация контента** | Средняя | 2–4 ч | Проверить, что «Пожаловаться»/«Заблокировать» заметны и функциональны. Добавить скриншоты функций модерации в заметки для ревью. |
| **Проблемы конфиденциальности** | Средняя | 2–4 ч | Проверить точность меток конфиденциальности. Убедиться, что URL политики конфиденциальности действителен и исчерпывающ. |
| **Соответствие шифрования** | Низкая | 1–2 ч | Предоставить документацию по шифрованию. При необходимости подать заявку на исключение CCATS. |
| **Проблемы с демо-аккаунтом** | Низкая | 1 ч | Убедиться, что демо-учётные данные работают. Предоставить пошаговые инструкции тестирования в заметках для ревью. |
| **Краши при ревью** | Низкая | 4–8 ч | Проанализировать логи крашей из App Store Connect. Исправить, протестировать, повторно отправить. |

### Распределение буферных дней

| День | Активность |
|------|-----------|
